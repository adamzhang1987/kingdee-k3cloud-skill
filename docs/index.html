整体架构

Architecture

Skill 与 MCP 的分工

Skill vs. MCP: division of labor

Skill（本项目） = 知识库 + 工作流决策树，让 Claude 自动掌握正确的 API 用法，避免字段名错误。MCP Server = 执行引擎，提供 15 个实际的 API 工具，适用于所有 MCP 客户端。两者可分别使用，但在 Claude Code 中组合使用效果最佳。

Skill (this project) = knowledge base + workflow decision trees, so Claude picks the right API usage automatically. MCP Server = the execution engine with 15 real API tools, usable from any MCP client. Best combined inside Claude Code.

Claude Code (Skill installed) │ Knowledge injection (form IDs, field names, workflows) ▼ Claude Code + MCP tools (query_bill_json, view_bill, ...) │ Kingdee Web API ▼ 金蝶云星空 K3Cloud
使用其他 AI 客户端？ 本 Skill 是 Claude Code 专属的知识增强插件。如果你使用 Claude Desktop、Cursor、Cline、Cherry Studio、Openclaw 等其他支持 MCP 协议的客户端，直接配置 kingdee-k3cloud-mcp 即可使用全部工具，无需安装本 Skill。
Using a different AI client? This Skill is exclusive to Claude Code. If you use Claude Desktop, Cursor, Cline, Cherry Studio, Openclaw, or any other MCP-compatible client, just configure kingdee-k3cloud-mcp directly — no Skill install needed.

从源码安装

Build from source

克隆仓库后运行 make build 打包，将生成的 .skill 文件复制到 skills 目录。

Clone the repo, run make build to package, then copy the resulting .skill file into your skills directory.

# 从源码安装Build from source
git clone https://github.com/adamzhang1987/kingdee-k3cloud-skill.git
cd kingdee-k3cloud-skill
make build
cp kingdee-k3cloud.skill ~/.claude/skills/

设计理念

Design Philosophy

Progressive Disclosure（渐进式披露）

Progressive Disclosure

主文件保持精简（<300 行），详细的字段验证表、查询模板、错误处理等放在 references/ 中按需引用。

The main file stays lean (<300 lines); detailed field tables, query templates, and error handling live in references/ and load on demand.
