# Contributing to kingdee-k3cloud-skill

Thank you for your interest in contributing!

## What This Project Contains

This is a Claude Code Skill — a collection of Markdown documents (`kingdee-k3cloud/`) that are packaged into a `.skill` file. No Python or build tools are needed.

## Making Changes

1. Fork the repo and create a feature branch from `main`
2. Edit files under `kingdee-k3cloud/` (SKILL.md or any file in `references/`)
3. The `.skill` file is auto-generated by CI on tag push — you don't need to rebuild it manually
4. Submit a pull request describing your changes

## Content Guidelines

- Keep `SKILL.md` concise (under 300 lines) — it's loaded on every skill invocation
- Put detailed references in `kingdee-k3cloud/references/` — they're loaded on demand
- Verify field names against a real K3Cloud instance before adding to `verified-fields.md`
- Examples should use clearly fictional data (e.g., `MAT001`, `CUST001`), not real business data

## Reporting Issues

Please open an issue with a clear description of the incorrect or missing information, including the K3Cloud version and form type if applicable.

## Changelog

If your PR adds a workflow, corrects field names, or makes a meaningful content change, please add a brief entry under the `[Unreleased]` section in `CHANGELOG.md`. Significant contributions will be credited by name in the release notes.

## Code of Conduct

This project follows the [Contributor Covenant v2.1](CODE_OF_CONDUCT.md). By participating, you agree to abide by its terms.
